# E4 — 프록시↔효능 캘리브레이션

`scripts/eval.js`의 구조 프록시는 가치 표면의 *존재*(=상한)를 잰다. E1 A/B(`eval/RESULTS.md` 1~4회차)는
실현 효능이 그 표면 지식의 **비국소성**에 비례함을 보였다. E4는 그 관계를 *모델*로 코드화하고 E1로 검증한다.

## 모델
```
실현델타(value) ≈ 비국소성계수(value) [0..1] × 표면존재(value) [0/1]
```
**비국소성 계수:**
| 가치 | 거처 | 계수 | 근거 |
|---|---|---|---|
| prewarning | `lessons/LSN-*` | **1.0** | 과거 반복 실패는 *어떤 코드 스냅샷에도 없다*(격자의 시간축). 정의상 비국소 |
| antiRegression | `project/HISTORY.md` | **1.0** | 과거 전략 결정도 코드에 없다(시간축). 정의상 비국소 |
| reuse | `reference/contracts/CON-*` | **구조 추정** | 정의 파일 밖에서 참조되는 CON id 비율(국소↔비국소가 코드 배치에 의존) |
| contextEconomy | (예산 가드) | 0.0 | 지식 *공급* 표면이 아니라 비용 가드 → 효능 델타에 직접 기여 안 함 |

> 시간축 두 계수가 상수 1.0인 것은 DESIGN_RATIONALE이 격자에서 *연역*한 사실의 직접 귀결이다 —
> 시간축 평면(§3.2, §4)은 *정의상* 단일 스냅샷에 없다. 계약만 구조 추정이 필요하다(코드 인라인 여부에 의존).

## E1 검증 (모델이 4개 측정점을 재현하는가)
| 측정점 | 계수(모델) | 예측델타 | 실측델타(E1) | 일치 |
|---|---|---|---|---|
| prewarning (T1 v2, region-pin) | 1.0 | 1.0 | +1.0 | ✓ |
| antiRegression (T3 v2, zeta) | 1.0 | 1.0 | +1.0 | ✓ |
| reuse — *인라인* 계약 (T2, 1회차) | 0.0 | 0.0 | 0 | ✓ |
| reuse — *비국소* 계약 (T4, 2·3회차) | 1.0 | 1.0 | +1.0 | ✓ |

→ **모델이 네 점 모두 재현.** 핵심 검증: 같은 가치(계약 재사용)인데 계수가 국소(0)↔비국소(1)로 갈리며
실측이 정확히 따라온다. 즉 프록시 상한만으로는 효능을 예측 못 하고, *비국소성 계수를 곱해야* 맞는다.

## 운용 — `scripts/eval.js`가 산출
`node scripts/eval.js`가 각 가치의 예측델타와 종합 예측효능(0~1)을 출력한다. 계약 계수는 평면에서
자동 추정(외부참조 CON 비율). 이로써 *모델 A/B 없이도* 평면 변경의 효능 영향을 연속 추적한다.

## 재실행 정책 (언제 A/B로 재앵커링하나)
프록시는 값싼 연속 신호, A/B는 비싼 앵커. 다음 중 하나면 A/B 재실행:
1. **계수 구조 변화** — 계약이 인라인화(외부참조 비율 급락)되거나, 새 평면 종류가 추가돼 계수 정의가 필요.
2. **프록시↔실측 괴리** — 마지막 측정 실효능과 종합 예측효능이 임계(예: 0.3) 이상 벌어짐.
3. **모델 티어 대폭 변경** — 단 E1-H4가 *모델 무관*을 보였으므로 우선순위 낮음.

## 정직한 한계
- 계약 계수는 *평면 참조* 기반 구조 추정이다 — *소비 코드*가 실제로 비국소인지(같은 파일 인라인인지)는
  평면만으로 완전히 알 수 없다. 보수적으로 "널리 참조되면 비국소"로 근사.
- [E1-H3] 토큰 순이득은 아직 모델에 없다(토큰 회계 필요). 다음: on의 선주입 비용 vs off의 재작업 비용을
  계량해 *순효능 = 효능델타 − 토큰비용*으로 확장.
- N 작음(E1). 계수 상수(1.0)는 강한 가정이나 시간축의 정의에서 직접 도출되어 신뢰도 높음.
