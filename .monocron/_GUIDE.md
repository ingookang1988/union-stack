# .monocron/ — 액자(경계) 계층 가이드

> **층위:** 액자(Boundary). 격자의 칸이 아니라 "이 프로젝트가 무엇인가"를 정하는 경계.
> **권한:** Schema (인간만 수정, 에이전트 read-only)
> **변화 속도:** 거의 불변

## 무엇을 싣는가
- **정체성:** 이 프로젝트가 푸는 문제, 존재 이유, 핵심 컨셉. (기술 백서 성격)
- **경계:** 무엇이 이 프로젝트의 스코프 안이고 밖인가. → 에이전트의 *경계 판별기*.
- **도메인 어휘:** 이 프로젝트만의 용어 정의. → 에이전트의 *어휘 사전*.

## 무엇을 빼는가
- 구현 세부, 작업 현황, 변하는 수치 → 다른 기둥으로.
- 마일스톤/일정 → `.roadmap/`.

## 사용법
- 세션 시작 시 **1회 주입**(온보딩). 매 작업마다 참조하지 않는다.
- 선기획·대량개발 방식이면 두껍게(풀스코프 가정), 린스타트업이면 얇게.

## 파일
- `IDENTITY_example.md` — 더미. 복제 후 실제 프로젝트로 교체.
