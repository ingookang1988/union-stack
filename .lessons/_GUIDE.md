# .lessons/ — 시간축 학습 평면 (제품 오답노트) 가이드  ★신규 평면
> **위치:** mechanism의 *시간축 짝*. mechanism이 사후 공간축 검증이라면,
>   이 평면은 *사전 시간축 경고*. **권한:** Wiki + 생명주기는 제안 기반.

## 싣는 것: 같은 계보에서 반복된 실패(디버그 케이스·도메인 함정).
## 주입 시점: Upward Fetching에 묶여 작업 진입 시 *사전* 경고로 로드.

## 고유 관리 규율 (가장 위험한 기둥 — 검증 안 된 정보가 프롬프트 맨 앞에 꽂힘)
1. 진입 장벽: 1회 실패는 교훈 아님. 2~3회 반복 관찰돼야 등재.
2. 만료: "왜 아직 유효한가" 근거와 결속. 근거 사라지면 은퇴 후보.
3. 스코프: 자기 ZFS 계보에만 매달림(무관 계보 힌트는 길을 잃게 함).
4. 비용 상한: 서사 아닌 경고. 한 줄 + ID 링크. 상세는 위임.

## 생명주기: 자동 은퇴(위험) / 수작업(북키핑 부활) 대신
##   에이전트 제안(→ .proposals) + 인간 승인으로 등재·은퇴.
## 파일: LSN-* (해당 계보 ID 공유). 예: LSN-01a_example.md
