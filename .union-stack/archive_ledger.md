- [2026-06-14][ADR-03]: v6.0 "Empirical Harness" 승격 — [PHASE-02]의 경험적 트랙(E1·E2·E4) 완료 근거. E1: 순수 비국소 측정 시 세 가치 +1.0·분산0, 모델-무관(haiku=sonnet=opus), 토큰 ROI ≈82×. E2: 100노드 평면 precision=recall=1.00·O(depth). E4: 캘리브레이션 모델(실현델타=비국소성계수×상한) E1 4점 재현. 근거: 아키텍처 주장이 *측정된 결과*로 전환됨(리뷰 필수보완 ②효능 eval 종결). 잔여: E3(enforce 도그푸딩)은 본질적 *종단*+권한 가림 작업으로 운영 트랙에 남김 — v6.0은 *경험적 증거* 확립을 표시하지 운영 단련 완료를 주장하지 않는다(정직한 경계).
- [2026-06-14][ADR-04]: 회귀 방지 원칙(E1 도출) — 시간축 평면(`reference/lessons`·`project/HISTORY`)을 "오버헤드"로 보고 제거 금지. 측정상 하네스 페이오프가 *가장 큰* 구간이 바로 비국소 지식(과거 실패·폐기 결정)이며, 그건 어떤 코드 스냅샷에도 없다. 근거: 미래 단순화 유혹에 대한 가드(eval/RESULTS.md E1 법칙).
- [2026-07-16][ADR-05]: [PRO-07] Fail-close 스코프 한정 승인·반영·검증 완주 — E5 해악 팔이 실측한 과잉 발동(H1 델타 −0.8: 전거 약한 양성 요청을 "모호함"으로 재분류)을 규칙 1 문언 한 문장("인간의 직접 요청 자체는 모호함이 아니다 — 규범·계약·HISTORY와 *충돌*할 때만 발동")으로 교정. 재측정: H1 델타 −0.8→0(5/5), T3 회귀 차단 5/5 유지(가드 무뎌짐 없음). 근거: eval/RESULTS.md E5·E5 후속. 의의: 측정→제안→승인→반영→재측정 자기진화 루프의 첫 실전 완주.
- [2026-07-24][ADR-06]: [E3] 운영 트랙 측정 완료 — ①의례 자발 수행률: union-stack 자기 레포 **0%**(편집 세션 2개 중 0), 로컬 전체 15%(27/183). 하네스 저자의 레포조차 산문 규칙만으론 의례를 수행하지 않음 → 훅(사전 강제)의 존재 이유가 자기 데이터로 확증. ②enforce 재생(hook-replay): 레포 내 122편집 중 **7건(6%) 차단, 전부 schema, lock 0**. 정탐 3(HISTORY·ARCH-00 — 실제로 승인 필요했고 승인 처리됨) 대 논쟁 4(_GUIDE 편집이 승인된 배치 중 파일 수만큼 반복 차단). 결정: **새 규칙·카브아웃 없이 enforce 전환 가능** — HOOKS.md에 이미 있는 "승인된 Schema 배치 중 mode를 warn으로 일시 하향" 운영 수칙으로 마찰 해소. 근거: eval/RESULTS.md §E3. 잔여: 라이브 훅 설치는 사용자 행위(에이전트 자동 설치 금지 불변식).
- [2026-07-24][ADR-07]: 계측기 오염 재발 — transcript-stats의 의례 판정이 임의 tool input을 스캔해 문서에 "blast-radius"를 *쓰는* Write/Edit을 수행으로 오집계(최초 100% → 수정 후 0%). 판정을 실행 표면(도구명·셸 명령·스킬 인자)으로 한정하고 회귀 3건 고정. 근거: 3회차 instrument-v2 교훈의 반복 — 자기 도메인 용어를 다루는 레포에서 문자열 매칭 계측은 항상 자기 언급에 오염된다(향후 계측기 설계 시 선점 확인 사항).
- [2026-07-24][ADR-08]: 런타임 훅을 **설치하지 않고 문서화만** 한다(기본 입장 확정). 근거: 하네스 권한 최소화 — 훅은 임의 명령 실행 + 도구 호출 거부라는 강한 권한이라, 커질수록 하네스가 새 실패 지점·마찰의 원천이 된다. 통제는 사후 게이트(permission-guard·린터) + 문서 규율 2층으로 충분히 다층이며 훅은 *선택적* 3층. 실측 뒷받침([ADR-06]): enforce 차단 7건 중 4건이 이미 승인된 배치의 반복 차단 = 강제력↑ → 마찰↑(비용은 매 세션 인간이 지불). 함의: E3의 두 측정 질문(①의례 수행률 0% ②차단 경제성 6%)은 답이 났고 라이브 전환은 *미완이 아니라 선택하지 않은 길*. 향후 세션은 "훅을 켜자"를 재제안하지 말 것 — 재검토는 마찰 비용 구조가 바뀔 때만(예: 배치 승인 단위 훅 지원).
