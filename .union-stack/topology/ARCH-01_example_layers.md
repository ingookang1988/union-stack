# [ARCH-01] (예시) 클린 아키텍처 레이어 규칙

## 의존성 방향 (예시)
`UI → App → Domain ← Infra` — 안쪽(Domain)은 바깥을 import하지 않는다.

## 테스트 전략 (층 1: 왜·어떻게)
- Domain 로직은 인프라 없이 단위 테스트한다.
- 모든 외부 의존은 포트로 추상화하고 어댑터를 mock한다.
- E2E는 이 경계에서만 돈다.
